# OpenClaw 2026.4.22

A large feature-and-fixes release with major provider expansion, better local and ACP ergonomics, stronger diagnostics, and a broad reliability and security sweep.

## Highlights

- xAI gained image generation, TTS, STT, and realtime transcription support
- Additional realtime or batch STT support landed for Deepgram, ElevenLabs, and Mistral
- The TUI now supports local embedded mode without a Gateway while still enforcing approval gates
- Onboarding can auto-install missing provider and channel plugins
- `/models add <provider> <modelId>` lets operators register models from chat without restarting the gateway
- Gateway diagnostics now export sanitized support bundles and enable stability recording by default
- GPT-5 behavior overlays were generalized across more providers and harnesses
- The bundled Tencent provider plugin and Mantle Claude Opus 4.7 support both landed in this release

## Fixes

- Better default thinking behavior and status reporting for reasoning-capable models
- Safer plugin install and update flows, config-include handling, and gateway config recovery
- Broad Codex, Claude CLI, ACPX, MCP, cron, Discord, Telegram, Slack, WhatsApp, and browser fixes
- Better protection against duplicated sends, stale auth profiles, stale sockets, and orphaned worker processes under memory pressure

## Breaking changes or operator-impactful changes

- Codex CLI auth import from `~/.codex` was removed from onboarding and provider discovery, so browser login or device pairing is now the supported path
- The legacy `OPENCLAW_CODEX_APP_SERVER_GUARDIAN` shortcut was removed in favor of `appServer.mode: "guardian"` or `OPENCLAW_CODEX_APP_SERVER_MODE=guardian`
- GPT-5 prompt-overlay personality is now managed through the shared provider runtime, with `agents.defaults.promptOverlays.gpt5.personality` as the main global toggle

## Relevance for this docs repo

This release is big enough to matter for troubleshooting and reference docs, especially around diagnostics exports, model registration, provider auth paths, Codex/ACP behavior, cron/MCP cleanup, and channel reliability.
