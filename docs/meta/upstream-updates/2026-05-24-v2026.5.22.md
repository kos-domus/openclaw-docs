# OpenClaw 2026.5.22

Stable `v2026.5.22` is now published on GitHub and npm. Local CLI remains on `2026.5.7`, so treat this as an upgrade-candidate release rather than an already-validated local baseline.

## Highlights

- Gateway startup and hot-path performance received a broad caching/lazy-loading pass across channel catalogs, plugin metadata, ACPX probes, and filesystem checks.
- Meeting Notes became a source-only external plugin with SDK source-provider contracts, manual transcript import, read-only CLI access, and Discord voice as the first live source.
- Realtime consults can now be queried, cancelled, steered, or queued from WebUI and Discord voice while a run is active.
- Image handling gained adaptive, model-aware compression through `agents.defaults.imageQuality`.
- Docs updates are unusually dense: Signal config paths, Telegram wildcard topic defaults, SecretRef guidance, cron HEARTBEAT behavior, gateway 403 troubleshooting, Codex migration notes, and multiple recovery runbooks.
- Packaging now excludes documentation images/assets from the npm tarball, reducing package size without dropping runtime docs search behavior.
- The reduced default sub-agent bootstrap context from the previous beta remains important: delegated workers should receive required persona/user/memory context explicitly when workflows depend on it.

## Breaking or operator-impactful changes

- Local installs on `2026.5.7` are behind the latest stable line; validate in staging before upgrading production bots.
- Workflows relying on implicit delegated-worker access to persona, user, memory, heartbeat, or setup files need explicit context handoff.
- Media understanding no longer auto-probes Gemini CLI first; configured provider APIs are preferred, with Antigravity CLI only as a lower-priority fallback.

## Fixes most worth caring about

- Gateway startup, plugin metadata, channel catalog, and Telegram session-store reads should be less noisy and cheaper.
- Cron, Telegram, Discord voice, WhatsApp recovery, and subagent-related docs/runbooks received practical operator cleanup.
- CLI/model auth handling improved for named provider profiles, especially Codex OAuth profile setup.
