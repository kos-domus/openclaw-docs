# OpenClaw 2026.5.10-beta.4

This beta is the first one in the `2026.5.10` train that looks operationally dense, not just experimental churn.

## Highlights

- New operator ergonomics: `/context map`, provider-level `localService` startup, leaner onboarding guidance, and more honest diagnostics around model transport and code-mode flows.
- Big Slack surface expansion: better DM/session canonicalization, reply-broadcast support, mention-source preservation, unfurl controls, and richer error visibility.
- Plugin SDK and Codex app-server keep moving in meaningful ways, especially around native tool ownership, grouped session actions, and structured media extraction.

## Breaking or operator-impactful changes

- `pnpm` 11 becomes the workspace package-management baseline.
- Codex app-server now firmly owns the native code/edit/exec tool path in that lane; treat this as migration terrain if you rely on older dynamic-tool assumptions.
- Private uploaded skill archives are now a real install surface, but only behind the explicit `skills.install.allowUploadedArchives` gate.

## Fixes most worth caring about

- Another real pass on Telegram and WhatsApp correctness, including WhatsApp `428 connectionClosed` retry handling and better Telegram reply/context behavior.
- Gateway and cron got useful truthfulness fixes: active manual cron runs stop looking `lost`, `gateway status --deep` stays plugin-aware, and health surfaces expose bootstrap degradation more honestly.
- Strong operator-grade cleanup landed around long transcript memory use, doctor/update safety, retry behavior, and Codex/OpenAI auth repair paths.
