# OpenClaw 2026.5.16-beta.7

This was an easy one to miss, but it matters: it is the last beta checkpoint before the `2026.5.18` stable line and it carries several operator-facing changes that later became part of the new stable gap.

## Highlights

- Raises the minimum supported Node.js 22 line to `22.19`, which is the biggest infra footgun in this stretch.
- Adds richer restart-trace attribution and a faster Gateway ready path by overlapping startup work more aggressively.
- Expands plugin and browser ergonomics with typed tool-plugin helpers and browser dialog control.

## Breaking or operator-impactful changes

- Any host, image, or CI path still pinned below Node `22.19` is officially behind this line.
- Local image customization now has a runtime-neutral apt build arg story (`OPENCLAW_IMAGE_APT_PACKAGES`), so infra docs should stop assuming the Docker-only legacy name.

## Fixes most worth caring about

- Better Telegram topic delivery safety, including fail-closed behavior when a topic thread no longer exists.
- Stronger subagent completion delivery and requester reconciliation across busy or stale parent runs.
- More durable config/plugin/update behavior instead of noisy partial recoveries.
