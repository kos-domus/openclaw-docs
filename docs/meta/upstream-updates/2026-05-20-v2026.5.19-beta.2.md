# OpenClaw 2026.5.19-beta.2

This is a tighter checkpoint on the same post-`2026.5.18` line: not a new direction, but a meaningful refinement pass before anyone should trust that train blindly.

## Highlights

- Keeps the Node `22.19` floor and the new image-build arg story (`OPENCLAW_IMAGE_APT_PACKAGES`) in focus, so infra drift is still the main operator footgun.
- Continues the runtime/accounting hardening theme with richer restart trace attribution and cleaner ACPX visibility.
- Still leans heavily on plugin/tooling cleanup, browser dialog handling, and subagent/channel delivery correctness instead of vanity features.

## Breaking or operator-impactful changes

- If a host or image pipeline still assumes older Node 22 minors, this line keeps making that debt harder to ignore.
- Restart diagnostics are richer now, which is good, but it also means operators should expect more detailed cost attribution in traces/logs.

## Fixes most worth caring about

- Better restart trace accounting across startup probe, config, runtime, and resource-count costs.
- More durable browser-dialog and plugin-tooling behavior in the same line as `2026.5.18` / `2026.5.19-beta.1`.
- Another bounded pass over delivery/runtime correctness rather than broad surface churn.
