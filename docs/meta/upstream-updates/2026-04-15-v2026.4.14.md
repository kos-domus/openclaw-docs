# OpenClaw 2026.4.14

A broad quality release centered on GPT-5.4 provider compatibility, channel/topic behavior, SSRF and allowlist hardening, and better reliability across subagents, cron, browser control, memory, and media flows.

## Highlights

- Forward-compat support for `gpt-5.4-pro` plus Codex catalog fixes and better GPT-5.4 reasoning compatibility
- Telegram forum topics now preserve human-readable topic names more reliably, including after restart
- Stronger security hardening for gateway config patching, browser SSRF enforcement, channel allowlists, and attachment path validation
- Reliability fixes for subagents, cron scheduling/backoff, browser CDP connectivity, delivery queue/session routing, and media handling
- Better support for proxy-backed installs, Ollama/OpenAI-compatible providers, WhatsApp media sends, and Google Gemini image generation

## Relevance for this docs repo

The release touches several documented surfaces that matter here: channel topic routing, security posture, cron behavior, subagent runtime reliability, browser configuration, and model/provider compatibility. Future doc refreshes should especially watch security guidance, Telegram/forum topic behavior, and provider reference coverage.
