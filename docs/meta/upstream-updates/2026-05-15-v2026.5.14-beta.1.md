# OpenClaw 2026.5.14-beta.1

Fresh beta line immediately after `2026.5.12` stable. Early read: still focused on plugin modularization, runtime correctness, richer UI/session behavior, and safer fallback/auth flows.

## Highlights

- New beta line starts by continuing the modular/plugin-externalization story instead of reversing course.
- Session/runtime visibility and rich-reply handling keep improving across Gateway, WebChat/TUI, and agent-to-agent flows.
- Auth and Codex/OpenAI startup behavior keep getting less brittle.

## Breaking or operator-impactful changes

- Early beta only. Useful to watch, not something to roll into production casually.

## Fixes most worth caring about

- More cleanup around session creation/bootstrap, reply delivery, and gateway protocol/session reporting.
- Continued hardening around config mutation serialization, plugin installs, and update safety.
- Keeps reinforcing the direction of a slimmer core runtime plus optional capability plugins.
