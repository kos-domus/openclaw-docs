# OpenClaw 2026.5.10-beta.1

First beta of the next line. Still noisy, but there is real operator signal in it.

## Highlights

- Better QA and diagnostics around Telegram PR evidence capture and Telegram Desktop scenario automation.
- Realtime voice work keeps expanding, especially on Discord voice plus `talk.realtime.instructions`.
- The Codex app-server line keeps getting more opinionated: dynamic tools are now firmly Codex-owned instead of split across older profiles.

## Breaking or operator-impactful changes

- Private skill archive upload is now a real capability, but explicitly gated behind `skills.install.allowUploadedArchives`. Good power, bigger attack surface if enabled casually.
- The Codex dynamic-tools profile goes away as a separate knob; Codex app-server is the default owner of workspace/edit/patch/exec/process/plan in that path.

## Fixes most worth caring about

- Telegram got another pass on no-response DM behavior and callback handling.
- OpenAI-compatible streaming, Anthropic-compatible reasoning replay, WhatsApp media upload, and Cron self-cleanup behavior all got concrete fixes.
- The security-adjacent catch worth remembering: a test-suite path had been able to clobber a real `~/.openclaw/exec-approvals.json`; upstream patched that host-leak footgun.
