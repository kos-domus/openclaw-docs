# OpenClaw 2026.4.11 upstream update summary

## Why it matters

Our installed version is `2026.4.8`, while upstream has moved to `2026.4.11`.

## Major additions

- Dreaming now supports ChatGPT import ingestion, plus new `Imported Insights` and `Memory Palace` diary subtabs for inspecting imported chats, compiled wiki pages, and source pages directly in the UI.
- Webchat/control UI now renders assistant media, reply, and voice directives as structured chat bubbles, adds `[embed ...]` rich output support, and gates external embed URLs behind config.
- `video_generate` gained URL-only asset delivery, typed `providerOptions`, reference audio inputs, per-asset role hints, adaptive aspect-ratio support, and a higher image-input cap.
- Feishu document-comment sessions got richer context parsing, reactions, and typing feedback.
- Microsoft Teams gained reaction support, reaction listing, Graph pagination, and delegated OAuth setup for sending reactions.
- Plugin manifests can now declare activation and setup descriptors for auth, pairing, and configuration flows.

## Fixes most relevant to our setup

- ACP parent streams no longer leak child assistant commentary during spawned runs.
- Explicit run timeouts are now honored more consistently by the LLM idle watchdog.
- Topic-scoped Telegram sessions keep the canonical transcript path when `MessageThreadId` is missing.
- WhatsApp reactions now use the gateway-owned action path, and default-account handling is more reliable.
- OpenAI/Codex OAuth no longer rewrites scopes in a way that breaks fresh sign-ins.
- QA/npm packaging fixes prevent repo-only QA assets from breaking packaged CLI startup or completion-state generation.
- Google Veo requests no longer send the unsupported `numberOfVideos` field.
- Failover/provider error surfacing is now scoped to the current attempt instead of stale history.

## Breaking-change watch

No hard breaking change is called out, but this release changes behavior in webchat rendering, plugin setup descriptors, ACP relay hygiene, timeout handling, and several channel integrations. Upgrade smoke tests should cover ACP/Codex flows, Telegram topic routing, WhatsApp reactions, and any plugin setup UX.

## Upgrade note

This is a meaningful maintenance release. For this repo specifically, the biggest value is cleaner ACP behavior, safer/fuller packaged CLI behavior, and tighter channel reliability while local install still lags by three releases.
