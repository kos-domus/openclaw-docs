# OpenClaw 2026.4.25

A broad operator-facing release focused on TTS, plugin cold-start/peristed-registry cleanup, richer diagnostics, safer browser/runtime behavior, and tougher install/update paths.

## Highlights

- Major TTS expansion: `/tts latest`, chat-scoped auto-TTS, personas, per-agent overrides, and new Azure Speech, Xiaomi, Local CLI, Inworld, Volcengine, and ElevenLabs v3 coverage
- Plugin startup/install paths move harder onto the persisted cold registry, reducing broad manifest scans and making plugin metadata handling more deterministic
- OpenTelemetry coverage expands across model calls, token usage, tool loops, harness runs, exec, outbound delivery, context assembly, and memory pressure
- Browser automation and Control UI both get a big polish pass, plus setup/install flows are tightened across Windows, macOS, Linux, and Docker

## Fixes that matter operationally

- WhatsApp keeps quiet-but-healthy linked sessions alive more reliably, with better voice-note/TTS handling too
- Bonjour/mDNS crash-loop handling is tightened, which matters directly for hosts that were seeing watchdog/restart churn
- Runtime context leakage and transcript hygiene got fixes (`sessions`, hidden runtime context, redaction, duplicate retries)
- Better ACP/Codex/subagent behavior, especially around completion delivery, startup probing, and thinking/model overrides
- Package update and bundled plugin runtime-dependency repair paths are much more defensive

## Breaking changes or operator-impactful changes

- Plugin install metadata is now expected in the managed plugin index rather than legacy authored config surfaces like `plugins.installs`
- Operators relying on old plugin/runtime assumptions should expect `doctor --fix`, update, and registry flows to rewrite metadata more aggressively
- The release continues the shift toward cold metadata paths and stricter startup/runtime separation, so custom plugin edge cases deserve smoke tests after upgrade

## Relevance for this docs repo

This is worth tracking even without a local upgrade yet: it touches WhatsApp reliability, Bonjour behavior, ACP/Codex ergonomics, plugin lifecycle, diagnostics, and TTS — all recurring themes in this repo's troubleshooting and reference docs.
