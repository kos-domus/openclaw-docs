# OpenClaw 2026.5.19

Stable moved again, and this is a real operator release, not fluff. The practical story is still runtime hardening, plugin/tooling maturity, and better restart/channel correctness.

## Highlights

- Stable upstream is now `2026.5.19`, so our local `2026.5.7` is materially behind the current floor.
- Node `22.19` is now the line to respect if we do any upgrade planning.
- Plugin/tooling surface got stronger with typed tool-plugin support, managed-skill global install/update, and native Codex plugin management from chat.
- Gateway startup/restart observability keeps improving with richer restart traces and better restart-ready timing.

## Breaking or operator-impactful changes

- Node `22.19` is the key compatibility constraint in this release train.
- Trusted admin HTTP RPC can now drive QR-login flows, which is useful but security-sensitive.
- Deprecation pressure is increasing around older plugin/message surfaces, so custom integrations should avoid legacy assumptions.

## Fixes most worth caring about

- Better Telegram topic isolation, retries, and fail-closed delivery behavior.
- More reliable subagent completion routing, pending-handoff persistence, and stale-session recovery.
- Continued Codex/OpenAI hardening around auth, compaction, tool replay, delivery, and context recovery.
- Better config/update/plugin repair paths, including cleaner handling of missing plugin packages and SecretRef-adjacent config writes.
