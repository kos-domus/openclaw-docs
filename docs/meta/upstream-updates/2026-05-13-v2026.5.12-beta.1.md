# OpenClaw 2026.5.12-beta.1

First drop of the `2026.5.12` beta line: broad again, but with a few immediately relevant operator moves.

## Highlights

- New `cron.get` / `openclaw cron get <id>` makes single-job inspection much less annoying.
- Per-sender tool policies are a real security/control upgrade for channel-routed fleets.
- `/context map`, provider-level `localService`, and ACP session-lineage exposure keep pushing operator visibility forward.

## Breaking or operator-impactful changes

- OpenAI login defaults further toward ChatGPT/Codex account auth; API-key setup is now the explicit alternate path.
- `pnpm` 11 remains part of the moving baseline in this beta line.
- Plugin SDK and Codex app-server are still shifting enough that plugin-heavy installs should treat this as migration terrain.

## Fixes most worth caring about

- Better WhatsApp 428 retry handling, long-transcript memory pressure relief, safer gateway/session lifecycle behavior, and more honest diagnostics.
- Useful cleanup across Telegram, cron, update/doctor flows, and Codex/OpenAI auth continuity.
