# OpenClaw 2026.5.16-beta.6

Fresh beta head, and this one is more than a tiny follow-up. It layers browser dialog handling, stricter Codex policy enforcement, Telegram long-poll durability, and more restart/subagent hardening on top of the already dense beta. 

## Highlights

- Browser automation gets a real dialog surface: pending modal visibility, `blockedByDialog` feedback, and explicit `browser dialog --dialog-id` handling.
- Codex policy enforcement is tighter: denied-tool turns now fail closed instead of accidentally leaving native code/app/MCP surfaces exposed.
- The beta keeps pushing plugin-first tooling and Mac/settings UX work, but the operational value is still in runtime correctness rather than shiny surface area.

## Breaking or operator-impactful changes

- Fail-closed Codex tool policy behavior is absolutely operator-impactful if you rely on mixed tool policies or native Codex paths.
- Gateway restart behavior, Telegram long-poll handling, and run-mode subagent registry retention all changed in ways that touch live operations and troubleshooting.

## Fixes most worth caring about

- Telegram isolated long polling now stays below the hard `getUpdates` request guard, which should reduce false disconnect/restart loops on idle accounts.
- Subagent lifecycle got another real pass: registry-save-before-accept, keep-mode visibility after sweeps, and same-origin completion routing on more channel paths.
- Gateway/config runtime got safer around in-process SIGUSR1/config restarts, protocol mismatch diagnostics, and local auth fallback behavior.
- Codex/OpenAI/Gemini replay paths kept hardening, including legacy OAuth migration tolerance and dropping malformed Gemini thought signatures before replay.
