# OpenClaw 2026.5.22-beta.1

This one is tag-only for now: the tag exists upstream, the mainline changelog has already opened the `2026.5.22` section, but there is still no matching GitHub prerelease entry or npm publish.

## Highlights

- Strong docs-heavy pass across onboarding, WhatsApp recovery, cron language/output behavior, skills, plugins, channels, SecretRef setups, and multiple operator troubleshooting paths.
- Default sub-agent bootstrap context got tighter: delegated workers now inherit only `AGENTS.md` and `TOOLS.md` unless extra context is explicitly passed.
- Provider/model listing performance got a real startup-path optimization via auth-state prewarm.
- Package integrity/security keeps getting stricter with shrinkwrap and locked dependency graph discipline.

## Breaking or operator-impactful changes

- The reduced default sub-agent context is the big behavioral shift here: workflows that quietly relied on `SOUL.md`, `USER.md`, memory, or other bootstrap files reaching workers by default can now behave differently unless context is passed on purpose.
- Media understanding now stops auto-probing Gemini CLI and treats Antigravity CLI as a lower-priority fallback after configured provider APIs.
- The old sender-owner tool-gating path is gone, continuing the move toward explicit trust/session identity instead of legacy compatibility shortcuts.

## Fixes most worth caring about

- More WhatsApp, Telegram, cron, and gateway/runtime reliability cleanup.
- Better package/update safety, plugin dispatch reuse, and managed-service/path correctness.
- Codex/OpenAI streaming, compaction, usage accounting, and tool-call integrity keep getting sharper.
- Session, delivery, and control-UI correctness continue to improve instead of drifting.
