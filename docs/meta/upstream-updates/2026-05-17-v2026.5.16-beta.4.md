# OpenClaw 2026.5.16-beta.4

This is the first beta in the line that feels strategically important, not just incremental. It adds real operator-facing surfaces: audit suppressions, parent-review subagent handoffs, xAI OAuth, cron wait semantics, restart tracing, and another large reliability pass.

## Highlights

- Adds `security.audit.suppressions`, which finally gives a sane way to accept known findings without polluting the active audit summary while still preserving them in JSON output.
- Subagent and delegated-task completions are now explicitly marked as ready for parent review, pushing requester agents to verify results before calling work done.
- Adds `openclaw cron run --wait` plus exact `cron.runs --run-id` filtering, which is immediately useful for automation and operational scripting.
- Resolved skill snapshots are cached across warm turns using the redacted effective config as the reuse key, reducing repeated skill rebuild overhead.

## Breaking or operator-impactful changes

- This beta touches security policy, cron semantics, auth, gateway restart tracing, group-chat routing, and subagent completion behavior — definitely not background noise.
- xAI OAuth for SuperGrok subscribers and the new group-chat `room_event` behavior both add meaningful new surfaces that deserve config review before blind adoption.

## Fixes most worth caring about

- Telegram kept getting sturdier: default-account preservation, captionless photo matching for catch-all groups, and isolated polling offset persistence are all practical fixes.
- Session and memory handling improved again, including background usage refreshes and incremental memory dirty-marking instead of heavy full rescans.
- Gateway approval, replay, and pairing paths got safer around symlinked exec approvals, malformed replay history, and loopback-origin validation.
