# OpenClaw 2026.5.16-beta.2

Mostly a hardening release, but not throwaway noise. It keeps pushing reliability across Telegram restarts, plugin/update repair paths, malformed persisted state recovery, and isolated cron fallback behavior.

## Highlights

- Telegram restart replay now persists queued polling updates so same-topic messages resume in order after a gateway restart.
- Isolated scheduled runs now honor configured subagent model fallbacks instead of quietly diverging under timeout failover.
- Codex/MCP scoping got cleaner with per-agent MCP server targeting and explicit native Codex approval-mode defaults.

## Breaking or operator-impactful changes

- Update and `doctor --fix` behavior keeps evolving around externalized plugins and channels, so package-swap repair logic is still an operator surface worth watching closely.
- Persistence hardening now aggressively skips malformed rows across sessions, cron, auth, pairing, commitments, and task sidecars; that is good for survival, but it can hide older corrupted state until you inspect it deliberately.

## Fixes most worth caring about

- Telegram `/stop` routing and control-lane handling got faster and less error-prone.
- Plugin and provider validation failures are more survivable during update/repair instead of taking the whole maintenance path down.
- A wide malformed-payload rejection sweep landed across media and provider integrations, which is boring in the best possible way.
