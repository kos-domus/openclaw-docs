# OpenClaw 2026.5.9-beta.1

Big beta, not a cosmetic one. It mixes a lot of platform churn with several genuinely useful operator-facing improvements.

## Highlights

- Better command/session ergonomics: `/think default`, `/fast default`, `openclaw path`, clearer CLI failure guidance, and `cron list --agent <id>`.
- More accurate runtime behavior surfacing: agents now get the actual provider/model identity in system prompts, and several UI/status surfaces got more truthful model/runtime reporting.
- Telegram, Discord voice/realtime, channel delivery, plugin SDK, and Gateway task/session internals all received broad work, so this beta touches many of the surfaces that matter in real fleets.

## Breaking or operator-impactful changes

- **BlueBubbles-backed iMessage is removed** in favor of `channels.imessage` via `imsg` on a signed-in Mac or SSH wrapper.
- **Node 22 floor is raised to 22.16+** (Node 24 still recommended).
- Plugin/channel/runtime delivery internals keep shifting, so this is a beta that absolutely deserves smoke tests before any upgrade.

## Fixes most worth caring about

- Telegram got more real fixes again: throttling is shared across polling and ad hoc sends, over-limit preview handling improved, pinned fallback transport can re-probe the primary path, private-topic sends fail correctly instead of falling back to DMs, and forum/tool-delivery edge cases keep getting cleaned up.
- A long tail of session/runtime correctness fixes landed: stale task reconciliation, safer session-store writes, transcript/session rotation fixes, gateway restart deferral escape hatch, better failover/cooldown handling, and fewer dropped final replies around tool/message flows.
- Docs/subagent bootstrap notes were corrected to include `SOUL.md`, `IDENTITY.md`, and `USER.md`, which is small but directly relevant for operator mental models.

## Relevance for this docs repo

Local install remains **behind stable upstream**: local `2026.5.2` vs stable `2026.5.7`.

This beta is still worth tracking because it touches several of our recurring pain points: Telegram topic delivery, session persistence, restart/recovery behavior, skill/plugin handling, and runtime diagnostics.
