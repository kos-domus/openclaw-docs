# OpenClaw 2026.5.5

Stability-heavy follow-up release focused on delivery correctness, session/UI recovery, provider compatibility, and safer operator repair paths.

## Highlights

- Messaging and interaction quality got another serious cleanup pass across Telegram, Discord, Slack, Matrix, and WhatsApp-facing runtime behavior.
- Gateway, TUI, and Control UI got responsiveness and recovery fixes that matter more than they look on paper, especially for large or messy session stores.
- Provider/runtime compatibility tightened around xAI reasoning controls, OpenAI-compatible streaming, Fireworks thinking behavior, and generated-media delivery.

## Breaking or operator-impactful changes

- `openclaw doctor --fix` introduced a risky Codex/OpenAI route rewrite in this release: it can rewrite valid `openai-codex/*` OAuth model routes to `openai/*`, which becomes important because `2026.5.6` immediately ships a revert/fix for that behavior.
- Plugin update syncing and official-plugin repair logic keep evolving in the npm-first plugin world, so fleets with pinned/disabled plugins should still smoke test updates instead of blind-bumping.

## Fixes most worth caring about

- Telegram/Codex progress drafts now stay visible without duplicating tool progress noise.
- Discord heartbeat timing and command authorization paths were corrected, reducing false reconnect loops and dropped plain-text control commands.
- `doctor --fix`, session recovery, TUI restore logic, and session cleanup all got safer around stale heartbeat-poisoned state.
- OpenAI-compatible SSE streaming now emits the assistant role chunk promptly instead of idling on a bodyless 200.

## Relevance for this docs repo

Local install stayed at **2026.5.2**, so this release widened the gap to three stable versions behind at detection time.

If upgrading from `2026.5.2`, the biggest things worth re-checking are Codex/OpenAI model routing, Telegram progress rendering, Discord stability, TUI/session recovery, and plugin update behavior.
