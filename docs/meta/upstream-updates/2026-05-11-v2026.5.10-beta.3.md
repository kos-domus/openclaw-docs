# OpenClaw 2026.5.10-beta.3

This is the beta in the series that starts to look strategically important, not just churn.

## Highlights

- New operator-facing ergonomics: `/context map`, background-process guidance, provider-level `localService` startup, and leaner prompt/tool-schema defaults.
- Slack got a big wave of real work: unfurl controls, reply broadcast, DM/session canonicalization, better file hydration, better room/thread behavior, and more truthful logs/errors.
- Plugin SDK and Codex app-server both moved again in meaningful ways, especially around native tool ownership, diagnostics, media extraction, and public SDK surface cleanup.

## Breaking or operator-impactful changes

- `pnpm` 11 becomes the workspace package-management surface.
- More Plugin SDK public subpaths are now deprecated or removed; plugin authors need to treat this beta as migration terrain, not as a patch release.
- Codex-native code-mode handling and new grouped session APIs are both powerful but invasive enough to justify smoke tests before any adoption.

## Fixes most worth caring about

- Telegram, WhatsApp, Slack, Control UI, Gateway restart behavior, Codex auth/rate-limit reporting, and OpenAI-compatible replay behavior all received substantial correctness work.
- Gateway health/status became more honest in a few important places: ACP backend readiness, model-pricing bootstrap degradation, and task completion routing.
- There is also another solid pass on Google/Gemini alias normalization, which matters because retired `gemini-3-pro-preview` ids keep resurfacing in the wild.
