## Operator-relevant highlights

This is a large release consolidating work from multiple streams. Key themes:

### 1. Reply delivery performance
- Visible reply delivery now separates user-facing sends from slower follow-up work
- Command/model/plugin metadata reused on hot paths
- Gateway startup avoids repeated plugin, channel, session, usage-cost, filesystem scans
- Telegram typing/progress context preserved during delivery

### 2. Voice & Talk maturity
- Realtime Talk: inspect, steer, cancel, follow-up from Web UI and Discord voice
- Wake-name handling more tolerant without ambient speech triggers
- Shared realtime SDK for turn-context, output activity, consult matching, activation-name matching

### 3. Channel production readiness
- **Telegram**: typing/progress context, forum topics, entity preservation, `ENETDOWN` recovery, targeted bot-command mentions, durable group retry
- **iMessage**: attachment roots, duplicate local Messages sources deduped, group media fixed, catchup cursor advancement
- **WhatsApp**: group/media behavior restored, thumb approval reactions
- **Discord**: voice playback improved, model picker bucket menus, wake matching tightened
- **Signal**: reaction approvals

### 4. Agent safety
- Codex app-server auth, compaction, source replies, sandbox path handling, usage-limit recovery more robust
- OpenAI-compatible providers: avoid empty-tool and malformed payload failures
- Session lock max-hold reclaim, stale lock ownership recovery
- Aborted subagent runs treated as terminal

### 5. Observability
- Activity tab (ephemeral, sanitized live tool activity)
- Gateway secret-prep traces
- Tool/model stream progress
- OpenTelemetry LLM content spans
- Alertable telemetry for blocked tools, failover, stale sessions, liveness, oversized payloads

### 6. Cron
- `cron.maxConcurrentRuns` defaults to 8 (parallel automation progress without explicit config)

### 7. Media
- Sharp replaced with Rastermill (no more Sharp install, smaller footprint)
- Adaptive model-aware image compression with `agents.defaults.imageQuality`
- EXIF normalization, HEIC/HEIF normalization before image descriptions

### 8. Memory
- Prevent semantic vector index silent degradation when embeddings unavailable
- CJK-aware dreaming dedupe
- Doctor OOM prevention on large session stores

### 9. Security hardening
- Session allowlist: linear matcher replaces backtracking-prone regex
- Transcript metadata field name escaping
- Plugin model-pattern regex validation
- Claude permission overrides audited under YOLO
- ACP auto approvals require explicit allow

### 10. Auth profiles
- Named model login profiles with credential migration for Hermes, OpenCode, Codex
- Explicit opt-out and non-interactive controls

## Notable fixes (operator impact)

- Telegram `/command@TargetBot` bot-command entities → explicit mentions for `requireMention` groups
- Telegram `ENETDOWN` → transient pre-connect network failure (same recovery as other outages)
- Codex: project newer OpenClaw chat history into resumed app-server threads; turn timeouts don't poison shared clients
- Config/doctor: narrow profiled tool-section repair, keep runtime-injected legacy web-search config out of user validation
- Windows: stack-size respawn for stack-heavy startup paths
- Stale plugin lock removal requires owner identity proof
- Usage: unknown all-zero model pricing shows as "missing cost entries" instead of confident `$0`
- IRC: canonical channel route storage, transient join before write
- QQ Bot: respect `OPENCLAW_HOME` for outbound media path resolution
