# OpenClaw 2026.5.7

Tight maintenance release, but not noise: it improves CLI/operator visibility, repairs a few real delivery and session edge cases, and hardens plugin/channel/runtime behavior.

## Highlights

- Better operator observability in the CLI: cron JSON now exposes computed job status, and channel listing is cleaner and more explicit about installed/configured/enabled state.
- More repair around long-lived sessions and delivery correctness: `/new`, `sessions.reset`, transcript rollover, and empty-success delivery paths all got sharper.
- Messaging/channel fixes matter for real fleets: Telegram polling/watchdog behavior, Telegram allowlists, Discord channel-target sends, Discord voice diagnostics, and WhatsApp proactive LID routing all improved.

## Breaking or operator-impactful changes

- No big breaking headline, but this release continues to touch `doctor --fix`, cron repair, channel/runtime surfaces, and approval flow behavior, so it still deserves a normal smoke test instead of a blind bump.
- It also extends the 2026.5.6 recovery path by preserving or repairing valid `openai-codex/*` OAuth routing when Codex auth is the real available path.

## Fixes most worth caring about

- `doctor --fix` now preserves working `openai-codex/*` PI routes and can repair 2026.5.5-broken `openai/*` GPT-5 routes when only Codex OAuth auth exists.
- `cron list --json` / `cron show --json` now expose computed status directly, which is genuinely useful for automation around job health.
- `openclaw channels list` became channel-focused and pushes auth/usage visibility to the more appropriate model/status commands.
- `/new` and `sessions.reset` now clear cached skill snapshots, avoiding stale skill lists in long-lived sessions.
- Delivery/session correctness improved in a few places: empty adapter sends no longer look successful, daily transcript rollover persists correctly, and `delivery.channel=last` fails before burning model tokens when there is no valid target.
- Telegram fixed sender allowlist handling and polling-watchdog liveness; WhatsApp improved proactive sends to LID-addressed contacts.

## Relevance for this docs repo

Local install is now **five stable releases behind upstream**: local `2026.5.2` vs upstream `2026.5.7`.

If we upgrade, the smart smoke-test set is: Codex OAuth routing, `web_fetch`, cron JSON/status tooling, Telegram delivery/allowlists, and any session flows that rely on `/new`, `sessions.reset`, or long-lived skill visibility.
