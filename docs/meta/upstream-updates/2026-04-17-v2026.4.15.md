# OpenClaw 2026.4.15

A large hardening release focused on provider defaults, bundled Google TTS, auth/status visibility, Codex and CLI runtime fixes, tighter tool and path security, and better reliability across channels, memory, cron, and replay flows.

## Highlights

- Bundled Google text-to-speech support, plus updated Anthropic and Claude CLI defaults
- New Control UI model-auth status card for OAuth health and provider pressure visibility
- Cloud-storage support for `memory-lancedb` and GitHub Copilot embeddings for memory search
- Stronger security around tool-name collisions, webchat audio/media embedding, Matrix pairing, `memory_get`, workspace file access, approvals redaction, and auth rotation behavior
- Broad reliability fixes across Codex, cron announce delivery, replay recovery, local-model prompt budgets, Telegram documents, WhatsApp reconnects, BlueBubbles catchup, and plugin cache invalidation

## Relevance for this docs repo

This release materially affects several areas already covered here: provider/auth setup, memory tooling, Codex and ACP behavior, cron announcements, security boundaries around file and media access, and channel integrations such as Telegram, WhatsApp, Matrix, and Discord.
