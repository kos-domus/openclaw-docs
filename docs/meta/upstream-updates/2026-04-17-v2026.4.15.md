# OpenClaw 2026.4.15

A feature-heavy release that expands speech, auth observability, memory storage, and local-model ergonomics, while also shipping a very large security and reliability sweep across tools, channels, replay, packaging, onboarding, and gateway behavior.

## Highlights

- Anthropic defaults now target Claude Opus 4.7, with refreshed aliases and bundled image-understanding defaults
- Gemini text-to-speech landed in the bundled Google plugin, including voice selection and telephony-friendly audio output
- Control UI now shows a Model Auth status card for OAuth health and rate-limit pressure
- `memory-lancedb` gained cloud-storage support for durable remote indexes
- Experimental `agents.defaults.experimental.localModelLean: true` reduces default tool weight for weaker local models
- Packaged builds are leaner thanks to plugin/runtime dependency cleanup and docs payload trimming
- Security hardening is broad: tool trust boundaries, media path validation, auth rotation, approval redaction, MCP loopback auth, workspace file safety, and QMD memory read restrictions all improved
- Reliability fixes span replay recovery, cron announce delivery, Codex/CLI sessions, startup reload loops, TUI streaming, BlueBubbles catchup, Telegram command sync, WhatsApp reconnect behavior, and provider failover classification

## Relevance for this docs repo

This release touches several areas already covered here: provider/auth configuration, TTS and speech flows, memory storage patterns, cron behavior, channel integrations, workspace-file safety, and Codex/ACP runtime behavior. The most likely docs needing refresh soon are provider/auth reference, local-model guidance, security/troubleshooting notes, and any pages that describe cron announce behavior or speech/TTS setup.
