# OpenClaw 2026.5.6

Small but very relevant hotfix release: it mainly undoes a bad `2026.5.5` doctor repair around Codex/OpenAI routing and cleans up fetch-path regressions.

## Highlights

- The main point of this release is operational safety: `doctor --fix` should stop "helpfully" breaking valid `openai-codex/*` OAuth model routes.
- Fetch/runtime handling got another cleanup so plugin HTTP requests and timed-out web fetches fail more cleanly instead of poisoning tool lanes.

## Breaking or operator-impactful changes

- No new breaking surface by itself, but this is effectively a recovery patch for `2026.5.5` fleets that may have had their default model or fallback routes rewritten incorrectly.
- Anyone who ran `doctor --fix` on `2026.5.5` should explicitly verify model routing after upgrade.

## Fixes most worth caring about

- Reverts the `2026.5.5` doctor repair that rewrote valid `openai-codex/*` ChatGPT/Codex OAuth routes to `openai/*`.
- Normalizes header dictionaries before native fetch/debug-proxy replay so plugin/runtime fetch paths stop failing on symbol metadata.
- Bounds timed-out web-fetch cleanup so request timeouts return a normal tool error instead of leaving Gateway tool lanes hanging.

## Relevance for this docs repo

Local install is now **four stable steps behind the latest upstream line**: local `2026.5.2` vs upstream `2026.5.6`.

If we upgrade through `2026.5.5`, verify Codex OAuth routing immediately after the bump and include `web_fetch` plus plugin-backed HTTP tools in smoke tests.
