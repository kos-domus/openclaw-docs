# OpenClaw 2026.5.16-beta.5

Big maintenance-heavy beta, but not empty noise. The real signal is around plugin tooling, Codex/OpenAI runtime correctness, restart/update safety, and a broad sweep of subagent and channel delivery fixes.

## Highlights

- Adds a typed simple-tool plugin workflow with `defineToolPlugin` plus `openclaw plugins build`, `validate`, and `init`, which matters if we keep pushing more functionality out into plugins instead of bloating core.
- Tightens bundled tool and skill metadata, shortens built-in tool descriptions, and refreshes CLI/API guidance without changing the guardrails.
- Adds several new bundled skills, but the operator-relevant part is really the continued investment in reusable local workflows and debugging surfaces.

## Breaking or operator-impactful changes

- This beta touches restart behavior, channel/plugin reload behavior, auth bootstrap, Codex/OpenAI routing, and subagent completion delivery. Not something to upgrade blindly in a busy production setup.
- The plugin/tooling additions are useful, but they also mean more movement in the plugin contract surface.

## Fixes most worth caring about

- Strong pass over subagent correctness: better parent-session completion handoff, keep-mode result persistence, sandbox ownership preservation, and fewer lost late completions.
- Better operational safety around updates and restarts: more platform-correct recovery hints, safer doctor/update plugin handling, and draining pending replies during gateway restart.
- Codex/OpenAI handling got sturdier again: compaction uses the real Codex context window, OAuth/bootstrap routing is less brittle, and oversized tool-result/context spillover is handled more safely.
- Telegram, WhatsApp, and channel delivery kept getting practical reliability fixes instead of cosmetic churn.
