# OpenClaw 2026.5.12-beta.5

This is a meaningful checkpoint in the same broad `2026.5.12` beta line, not a brand-new direction.

## Highlights

- More operator-facing hardening landed around approvals, transcript redaction, plugin install behavior, and agent fallback when model streams stall.
- Telegram and WhatsApp both got practical reliability work again, especially around polling, token rotation, formatting preservation, and close/debounce behavior.
- The beta still carries important Codex/OpenAI migration fixes, plus more Gemini 3.1 id normalization cleanup.

## Breaking or operator-impactful changes

- Still a high-churn beta touching auth, packaging, plugins, channels, and gateway protocol behavior.
- Worth smoke-testing, not blind-upgrading, especially if we care about Codex migration paths, plugin lifecycle, or messaging-channel stability.

## Fixes most worth caring about

- Better fallback escalation for silent model streams, better shared config mutation safety, and stronger transcript redaction coverage.
- Good channel-side fixes too: Telegram long-poll/token-rotation recovery, HTML preservation, WhatsApp reconnect/media-size handling, and cleaner auto-reply failure visibility.
- Net read: beta.5 looks like a serious hardening pass on the same line, with real operator value but the same beta risk profile.
