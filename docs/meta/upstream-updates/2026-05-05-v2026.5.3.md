# OpenClaw 2026.5.3

Large operator-facing release centered on npm-first plugin lifecycle hardening, gateway hot-path performance, delivery/streaming behavior, and a long tail of runtime/channel fixes.

## Highlights

- Bundled a new `file-transfer` plugin with binary-safe remote file tools (`file_fetch`, `dir_list`, `dir_fetch`, `file_write`) guarded by explicit per-node path policy, approval, symlink refusal by default, and a 16 MB transfer cap.
- Hardened official plugin install/update/uninstall/onboarding flows for the npm-first external-plugin world, including stale state repair, trusted catalog handling, and better beta/stable fallback behavior.
- Cut gateway startup/runtime overhead by lazy-loading plugin discovery, cron, schema metadata, shutdown hooks, and other hot-path work only when needed.
- Expanded delivery/streaming behavior with shared `streaming.mode: "progress"` previews and better channel recovery across Telegram, WhatsApp, Discord, Slack, Teams, Matrix, and Feishu.
- Added `/steer <message>` for queue-independent steering of the active run and `/side` as an alias for `/btw`.

## Breaking or operator-impactful changes

- Gateway config now fails closed on invalid config during startup/hot reload; automatic last-known-good restore is gone, and `openclaw doctor --fix` becomes the repair path.
- `tools.deny: ["write"]` no longer implicitly blocks `apply_patch`; operators must deny `apply_patch` or `group:fs` explicitly if they want patch writes blocked too.
- Official plugin management is now much more package/provenance-aware, so old assumptions around bundled artifacts, stale install ledgers, or manual plugin path drift are less safe.

## Fixes most worth caring about

- Telegram finals/topics delivery reliability, topic command behavior, media-group tuning, and stale reply suppression.
- WhatsApp inbound/output reliability, newsletter/channel targets, native mentions, and dependency install edge cases.
- `sessions_send` delayed replies surviving wait-window timeouts, cron runtime-state repair, and gateway restart/update safety.
- Memory/plugin/runtime diagnostics got materially clearer, especially around doctor repair, memory slot/plugin state, and large session stores.

## Relevance for this docs repo

Local install is now **one release behind** upstream stable: local `2026.5.2` vs upstream `2026.5.3`.

Highest-value smoke tests after upgrade: plugin lifecycle flows, Telegram topic delivery, WhatsApp routing/mentions, `sessions_send` delayed replies, and any config-repair workflow that currently leans on permissive gateway auto-recovery.
