# OpenClaw 2026.4.20

A large maintenance release focused on safer agent defaults, session-store pruning, cron durability, Kimi defaults, pairing visibility, and broad reliability/security fixes across exec, plugins, Codex, browser, Telegram, cron, and BlueBubbles.

## Highlights

- Stronger default system prompts and GPT-5 overlays, plus safer completion and verification guidance
- Session-store pruning and entry caps now apply by default, reducing the risk of gateway OOM from backlog growth
- Cron runtime state moved into `jobs-state.json`, keeping `jobs.json` cleaner for git-tracked definitions
- Better pairing diagnostics, scope-upgrade visibility, and device approval UX in doctor and Control UI
- Broad fixes across Codex OAuth/Responses routing, YOLO exec, plugin runtime dependency repair, browser connectivity reporting, Telegram polling/delivery, and channel reliability

## Relevance for this docs repo

This release matters for fleet operators running cron-heavy or multi-agent setups. The session-pruning behavior, cron state split, prompt/default changes, pairing diagnostics, and Codex/runtime fixes all touch areas already documented here.
