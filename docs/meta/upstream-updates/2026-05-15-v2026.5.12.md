# OpenClaw 2026.5.12

Big stable drop, not a tiny patch. The release clusters around plugin/install hardening, security and transcript redaction, better Telegram durability, better fallback behavior, and the continuing split of optional surfaces out of the core runtime.

## Highlights

- Core install/update paths got safer: plugin/runtime scanning tightened, pnpm 11 install edges were fixed, and update/doctor paths are less likely to leave managed installs in a weird half-state.
- Messaging reliability improved in places we actually care about: Telegram polling stall detection, token-rotation offset recovery, HTML rendering, and multiple WhatsApp reconnect/install fixes.
- Runtime/operator quality moved too: idle-watchdog fallback escalation, safer transcript redaction, better OpenAI-compatible validation, and more honest gateway/session diagnostics.

## Breaking or operator-impactful changes

- This is a real operator release, not background noise. It touches plugins, auth, approvals, Telegram/WhatsApp, Codex/OpenAI migration, gateway behavior, and security policy surfaces.
- Local CLI is now behind stable upstream: local `2026.5.7` vs upstream `2026.5.12`.

## Fixes most worth caring about

- Better plugin install and update safety, including fewer false failures around managed npm/plugin dependency handling.
- Telegram got sturdier around dead polling detection, legacy offset cleanup after token rotation, and formatted HTML reply rendering.
- Fallback behavior improved: silent model stalls escalate through profile rotation / configured fallback instead of just hanging.
- Transcript and tool-result persistence got stronger redaction, which matters for long-lived multi-agent logs.
- WhatsApp kept moving out of the heavy core bundle direction while also getting retry/reconnect fixes that reduce brittle setup behavior.
