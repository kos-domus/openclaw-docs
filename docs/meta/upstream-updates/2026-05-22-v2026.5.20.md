# OpenClaw 2026.5.20

Stable moved again, and this one is worth logging properly because it adds real operator surface instead of random churn.

## Highlights

- Stable upstream is now `2026.5.20`, so our local `2026.5.7` is even further behind the current stable line.
- The bundled Policy plugin is now part of the story for policy-backed channel conformance checks, doctor lint findings, and opt-in workspace repair.
- xAI gained device-code OAuth, which matters for remote or headless setups.
- Discord voice/realtime and bounded bootstrap-context behavior keep getting more deliberate instead of ad hoc.

## Breaking or operator-impactful changes

- The old skill-wrapper auto-allow compatibility path is gone: skill files must be loaded with the `read` tool, and only the real skill executable is auto-allowed.
- The Codex harness moved again (`@openai/codex` `0.132.0`), so beta/stable users should still expect runtime churn around that surface.
- `agents.list[].experimental.localModelLean` adds more per-agent config flexibility, but also another place where fleet policy can drift if left unreviewed.

## Fixes most worth caring about

- Continued WhatsApp/Baileys maintenance plus delivery-drain cleanup.
- More reliable cron correctness and wake-lane behavior.
- Better subagent completion handoff and recovery paths.
- Ongoing hardening around config writes, plugin trust, and provider/runtime routing.
