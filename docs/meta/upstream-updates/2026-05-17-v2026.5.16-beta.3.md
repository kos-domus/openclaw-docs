# OpenClaw 2026.5.16-beta.3

This beta is about correctness and operator visibility more than shiny features: safer diagnostics, tighter session scoping, better Telegram targeting, and fewer lost completions or weird timeout hangs.

## Highlights

- Credential-bearing gateway diagnostics are redacted more carefully without losing the raw programmatic data paths operators still need.
- Telegram scheduled announce targets now resolve through the same session registry as inbound routing, which should reduce drift between automation and live chats.
- Session bookkeeping got cleaner: agent-scoped session lookup, more honest local transcript usage accounting, and less stale metadata after recreation/compaction.

## Breaking or operator-impactful changes

- Gateway restart/shutdown handling keeps getting touched; if you rely on update churn or hot restarts, this beta line is relevant.
- Codex runtime timeout/liveness behavior changed again, especially around approvals and long-running turns, so timeout assumptions may need revalidation.

## Fixes most worth caring about

- Telegram startup identity caching and final-answer recovery both improved, which matters for flaky API periods and progress-mode chats.
- Update flows preserve channel config more reliably through package-swap repair and externalized plugin recovery.
- Auth/profile writes are more serialized, reducing OAuth credential races that can overwrite fresh state with stale in-memory snapshots.
