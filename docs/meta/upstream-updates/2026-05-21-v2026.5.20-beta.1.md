# OpenClaw 2026.5.20-beta.1

Fresh beta head right after the `2026.5.19` stable cut. Same general direction, but with a few new bits worth watching before they land downstream.

## Highlights

- Adds the bundled Policy plugin for policy-backed conformance checks and doctor linting.
- Brings xAI device-code OAuth, which is handy for headless or remote setups.
- Pushes more Discord voice/realtime work, including user-follow voice behavior and bounded profile bootstrap context.

## Breaking or operator-impactful changes

- The Codex harness moved again (`@openai/codex` `0.132.0`), so anyone riding the beta line should expect continued runtime churn.
- More config surface is becoming agent-scoped (`experimental.localModelLean`), which is useful but worth watching for policy drift.

## Fixes most worth caring about

- WhatsApp got a Baileys bump plus a real outbound-delivery drain fix, which matters if reconnect timing has been sketchy.
- Cron got several correctness fixes around wake lanes, success/failure classification, and legacy store compatibility.
- Subagent completion recovery and requester handoff keep getting tightened instead of hand-waved.
- Status/model pin visibility improved, which should make stuck session-model mismatches less opaque.
