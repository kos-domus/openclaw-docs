# OpenClaw 2026.4.26

A feature-heavy release with realtime voice/browser work, new migration tooling, another provider expansion, and a deep reliability sweep across startup, plugins, cron, subagents, and local/self-hosted providers.

## Highlights

- Talk mode gains a generic browser realtime transport plus Google Live browser sessions and Gateway relay support for backend-only realtime voice plugins
- New bundled provider: Cerebras
- New `openclaw migrate` flow plus bundled importers for Claude and Hermes configurations
- Raw config diffing in Control UI gets much better, with safer redaction and cleaner pending-change inspection
- Compaction gains an opt-in transcript-bytes preflight trigger for oversized active JSONL sessions

## Fixes that matter operationally

- `sessions_spawn` now respects configured/requested child model overrides earlier and more reliably
- Explicit `runtime="acp"` spawns are fixed even when automatic ACP dispatch is disabled
- Cron/model enforcement is stricter: explicit cron payload models now fail closed instead of silently drifting to another model
- WhatsApp QR-login WebSocket traffic now honors proxy env vars
- Bonjour now defaults mDNS advertisements to the system hostname when safe, reducing `.local` conflicts and restart churn on some hosts
- Plugin install/discovery/runtime paths keep getting safer around symlinks, staged deps, packaged installs, and degraded startup behavior

## Breaking changes or operator-impactful changes

- Plugin config helpers are being deprecated in favor of runtime snapshots plus transactional mutation helpers
- More bundled behavior is now manifest-owned (model catalogs, startup activation, provider metadata), so plugin authors should expect less implicit core fallback behavior over time
- The release continues fail-closed behavior on model resolution and cron execution, which is good for safety but can surface latent misconfigurations immediately after upgrade

## Relevance for this docs repo

This one matters for fleet ops. The local install is still on 2026.4.24, so we're missing fixes around subagent model overrides, ACP spawn behavior, cron model enforcement, proxy-aware WhatsApp login, and additional startup/plugin hardening.
