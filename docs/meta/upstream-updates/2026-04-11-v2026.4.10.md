# OpenClaw 2026.4.10 upstream update summary

## Why it matters

Our installed version is `2026.4.8`, while upstream has moved to `2026.4.10`.

## Major additions

- Bundled Codex provider and plugin-owned app-server harness, so `codex/gpt-*` can use Codex-managed auth, native threads, model discovery, and compaction separately from the standard OpenAI provider path.
- New optional Active Memory plugin that injects a dedicated memory sub-agent before replies, with configurable context depth, `/verbose` inspection, and optional transcript persistence.
- Experimental local MLX speech provider for macOS Talk Mode, with interruption handling and fallback to system voices.
- New `openclaw exec-policy` command to inspect and sync local exec approval settings.
- Gateway now exposes `commands.list` so remote clients can discover native commands, skills, and plugin commands with structured metadata.

## Fixes most relevant to our setup

- Stronger browser and sandbox SSRF defenses, navigation checks, CDP hardening, and media/file boundary enforcement.
- Broader security hardening around exec preflight reads, host env denylisting, profile mutations, plugin installs, Gmail watcher token redaction, and websocket frame handling.
- Better gateway startup behavior so websocket RPC and remote access come up earlier even when channels or sidecars stall.
- Thread and route preservation fixes across Slack, Telegram, Mattermost, Matrix, ACP, cron, and subagent announcements.
- Cron and agent reliability fixes, including invalid timestamp self-healing, auth-profile consistency, timeout handling, and heartbeat timeout configuration.
- Skill/plugin loading fixes so bundled skills remain discoverable and user-home overrides work correctly.

## Breaking-change watch

No single headline breaking change is called out, but there are meaningful behavior shifts around Codex provider routing, Active Memory, exec-policy management, and stricter security validation. Any setup relying on old provider assumptions, permissive network access, or loose plugin/config behavior should be smoke-tested after upgrade.

## Upgrade note

This looks like a strong maintenance upgrade, especially for environments using ACP/Codex, cron jobs, remote gateway access, browser tooling, or multi-agent routing.
