# OpenClaw 2026.5.14-beta.2

Dense beta checkpoint with real operator signal: bootstrap/profile controls, auditable subagent task delivery, WhatsApp status reactions, and a long reliability pass across Telegram, cron, doctor, and plugin-backed channels.

## Highlights

- Adds per-agent bootstrap-profile overrides for context injection and bootstrap size limits, which matters directly for large `SOUL.md` / `AGENTS.md` setups.
- Makes native `sessions_spawn` tasks visible in the child session's first message instead of hiding them in the system prompt, improving delegation auditability.
- Extends lifecycle/status reactions to WhatsApp message turns, aligning it better with Telegram and Discord.

## Breaking or operator-impactful changes

- Removes the bundled `codex-cli` backend and repairs legacy `codex-cli/*` model references onto the Codex app-server route on `openai/*`.
- `doctor --fix` and status surfaces got sharper around plugin/channel corruption and stale auth shadows, so older dirty installs should be expected to show more explicit warnings.

## Fixes most worth caring about

- Telegram got meaningful hardening around bot initialization, method-aware timeouts, independent topic/DM draining, and stuck polling recovery.
- Cron timeout semantics are less wrong now: once an isolated run actually starts, it follows the configured job timeout instead of the earlier pre-execution watchdog behavior.
- Multiple channel/plugin/status paths now fail louder and cleaner instead of silently looking healthy when they are not.
