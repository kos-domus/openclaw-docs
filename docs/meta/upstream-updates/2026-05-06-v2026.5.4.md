# OpenClaw 2026.5.4

Follow-up stable release after `2026.5.3` / `2026.5.3-1`, focused on Google Meet voice flow quality, plugin/package lifecycle repair, gateway hot-path performance, and a very large batch of runtime/channel fixes.

## Highlights

- Google Meet / voice-call path got materially better: agent-mode talk-back is now the default direction, Twilio dial-in joins use a tighter realtime voice bridge, and backpressure/queue handling was hardened to reduce lag and self-interruption.
- Plugin/operator ergonomics keep improving in the npm-first world: better install hints for missing official plugins, more trusted migration/update behavior, and cleaner repair around stale bundled/external plugin state.
- Gateway and agent hot paths got another performance pass, especially around startup, plugin metadata reuse, session listing, tool registration, and config-only runtime snapshots.
- Release also carries a broad reliability sweep across Telegram, WhatsApp, Discord, Slack, Codex/OpenAI auth/media, web fetch/search, cron/session handling, and approval/tooling surfaces.

## Breaking or operator-impactful changes

- No single headline breaking change like `2026.5.3`'s fail-closed config startup, but the release keeps tightening plugin discovery/install rules and official-plugin ownership behavior; that is worth a smoke test on fleets with custom plugin layouts.
- Google Meet semantics shifted further toward explicit `agent` vs `bidi` behavior, so anyone actively using Meet/Twilio joins should re-check expected mode/config after upgrade.

## Fixes most worth caring about

- Telegram forum-topic routing, button delivery, streaming cleanup, media placeholder correctness, and active-chat status targeting.
- WhatsApp login/output routing, allowlist canonicalization, newsletter targets, and group reply behavior.
- Plugin update/install repair for official external packages, correction-version handling (`2026.5.3-1` style), and stale shadow cleanup.
- Cron/session diagnostics, context-overflow loop guarding after compaction, and large session-store bounding/perf fixes.
- Windows/macOS service/update/path handling, plus extra SSRF/Windows env hardening.

## Relevance for this docs repo

Local install is now **two stable steps behind the latest upstream line**: local `2026.5.2` vs upstream `2026.5.4`.

If we upgrade, the highest-value smoke tests are: plugin install/update flows, Telegram topic delivery, WhatsApp routing/login, Google Meet voice mode behavior, and any cron/session workflow that previously hit context-overflow or large-store slowdowns.
