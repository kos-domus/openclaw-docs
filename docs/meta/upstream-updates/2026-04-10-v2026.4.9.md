# OpenClaw 2026.4.9 upstream update summary

## Why it matters

Our installed version is `2026.4.8`, while upstream has moved to `2026.4.9`.

## Major additions

- Memory and dreaming now support grounded REM backfill, diary commit/reset flows, cleaner durable-fact extraction, and live short-term promotion.
- Control UI adds a structured diary view with timeline navigation, backfill/reset controls, grounded Scene visibility, and traceable dreaming summaries.
- QA tooling adds character-vibes evaluation reports with model selection and parallel runs.
- Provider manifests can declare `providerAuthAliases` so provider variants can reuse auth flows and env vars.
- iOS release versioning is pinned to explicit CalVer with a documented release-train workflow.

## Fixes most relevant to our setup

- Browser safety checks now re-run after interaction-driven navigations, closing an SSRF bypass path.
- Untrusted workspace `.env` files can no longer inject runtime-control or browser-control override variables.
- Remote node exec event summaries are treated as untrusted and sanitized before later turns.
- Explicit auth-profile upserts now persist directly instead of depending on stale external CLI sync.
- Agent idle timeout behavior is more predictable, including safer cron defaults.
- Session routing preserves established external routes, reducing cross-channel delivery regressions.
- Internal control tokens like `ANNOUNCE_SKIP`, `REPLY_SKIP`, and stray `NO_REPLY` are less likely to leak into user-visible replies.
- `openclaw doctor` and reply-run auth flows surface reauth issues more clearly.

## Breaking-change watch

No obvious hard breaking changes are called out in the release notes, but this release tightens security and auth handling. Any workflow that depended on workspace `.env` overrides or loose browser-control settings should be rechecked after upgrade.

## Upgrade note

This looks like a worthwhile maintenance upgrade for environments using cron runs, remote nodes, auth profiles, browser automation, and multi-channel session routing.
