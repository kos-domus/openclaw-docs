# OpenClaw 2026.5.19-alpha.1

The alpha line opened immediately after `2026.5.19-beta.2`, so this is early-signal material, not upgrade bait. Still, it exposes where upstream is pushing next.

## Highlights

- Gateway startup keeps getting faster and less serialized by overlapping plugin-service startup with channel sidecars while preserving readiness gating.
- Trusted admin HTTP RPC clients can now initiate and wait through web QR login flows, which matters for operator tooling and remote recovery.
- Skills and CLI surface keep expanding (`--global` managed skill install/update, renamed `autoreview`, new meme-maker), but the real value is still in runtime-control ergonomics.

## Breaking or operator-impactful changes

- This is alpha. Treat it as directional intelligence, not a candidate for casual deployment.
- Admin HTTP RPC gaining QR-login control is powerful and worth reviewing carefully in any trusted-client setup.

## Fixes most worth caring about

- Faster gateway ready path via overlapping startup work.
- Better remote/admin recovery paths for QR-login-dependent flows.
- Continued bounded maintenance across browser/runtime/skills surfaces without changing the core stable-gap story for our local install.
