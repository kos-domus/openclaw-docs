# OpenClaw 2026.5.16-beta.1

Fresh beta line focused less on shiny surface area and more on runtime correctness: skill-cache reuse, cron fallback fidelity, Telegram recovery, and safer provider/config parsing across a lot of edges.

## Highlights

- Caches hydrated `resolvedSkills` across warm turns using the redacted effective config as the reuse key, which should cut repeated skill rebuild overhead without crossing config boundaries.
- Adds opt-in Telegram group `ambientTurns: "room_event"`, useful for quiet room-context ingestion where only message-tool output should speak visibly.
- Keeps pushing onboarding/localization and plugin-first ecosystem cleanup rather than adding random one-off core hacks.

## Breaking or operator-impactful changes

- Cron behavior keeps changing in meaningful ways: model fallback policy, external-channel plugin loading, transcript mirroring, and failure-delivery accounting all moved in this beta.
- Codex/MCP scoping got stricter and cleaner, including agent-scoped MCP exposure and explicit native approval defaults.

## Fixes most worth caring about

- Telegram got another serious reliability pass: reconnect queue draining, ingress-health detection when backlog wedges, stronger partial/final reply preservation, and active-memory routing fixes.
- Scheduled isolated runs now respect configured subagent fallback policy instead of silently diverging under timeout failover.
- A lot of malformed JSON / malformed payload handling was hardened across providers, channels, and export paths, which is boring but exactly the kind of boring that prevents stupid production crashes.
