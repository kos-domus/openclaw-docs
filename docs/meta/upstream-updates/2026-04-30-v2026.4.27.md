# OpenClaw 2026.4.27

## Highlights

- **Codex Computer Use**: new `/codex computer-use status/install` commands with marketplace discovery, optional auto-install, and fail-closed MCP server checks before Codex-mode desktop control turns start.
- **DeepInfra provider**: bundled provider with `DEEPINFRA_API_KEY` onboarding, OpenAI-compatible model discovery, image gen/editing, TTS, text-to-video, and memory embeddings.
- **Tencent Yuanbao and QQBot**: Yuanbao external channel plugin for WebSocket bot DMs and group chats; QQBot gains full group chat support with history tracking, @-mention gating, streaming, chunked media upload, and pipeline refactors.
- **Plugin manifest-first catalogs**: model catalog rows, aliases, and suppressions now declared in plugin manifests, reducing Gateway boot work.
- **Operator-managed outbound proxy**: opt-in `proxy.enabled` + `proxy.proxyUrl` / `OPENCLAW_PROXY_URL` with strict forward-proxy validation and loopback-only Gateway bypass.

## Breaking / operator-impactful

- Plugin startup now uses explicit `activation.onStartup` manifest declarations; deprecated implicit startup sidecar loading emits compatibility warnings.
- Bundled agent tool-result middleware loads from manifest contracts on demand (tokenjuice stays startup-lazy).
- Plugin SDK testing: many repo-only `test/helpers` bridges retired in favor of focused SDK test subpaths (`plugin-sdk/channel-test-helpers`, `plugin-sdk/agent-runtime-test-contracts`, etc.).

## Security

- Operator-managed outbound proxy routing with strict http:// forward-proxy validation.
- Plugin SDK generic host hooks for session state, trusted tool policy, and run-scoped plugin context.
- Matrix: versioned structured approval metadata on pending approval messages.

## Fixes

- Gateway sessions/history: thinking defaults aligned with owning-agent and catalog-aware resolution.
- Device pairing: array-shaped pairing state files recovered before persisting approvals.
- Gateway auth: stale device tokens cleared to avoid rate-limit loops after token rotation.
- Telegram: `approvals.exec/plugin.targets[].accountId` honored for multi-bot approval routing; outbound Bot API calls bounded to prevent gateway wedges.
- Slack: 15s pong timeout default with configurable overrides; stalled `file_share` media downloads bounded.
- Auto-reply: progress-aware idle timeout for post-run pending tool-result delivery drain.
- Gateway startup: chat channels start without waiting for primary model prewarm.
- CLI update: tracked plugins disabled in config skipped during post-update plugin sync.
- CLI status: scope-limited gateway probes treated as reachable-but-degraded.
- Media tasks: detached `video_generate` / `music_generate` contexts kept live; OpenAI media providers preferred when using Codex auth alias.

## SDK / Plugin ecosystem

- Plugin SDK: shared channel route normalization, parser-driven target resolution, and route comparison helpers via `openclaw/plugin-sdk/channel-route`.
- Plugin SDK: generic host hooks for session state, next-turn context, trusted tool policy, UI descriptors, events, scheduler cleanup, and run-scoped plugin context.
- Plugin SDK testing: broad migration from repo-only bridges to focused SDK test subpaths.
- Plugin SDK: bundled-channel schema SDK surface added; deprecated legacy config schema subpath tracked with removal window.

## Dependencies

- Refreshed AWS SDK, PI runtime, AJV, Feishu SDK, Anthropic SDK, tokenjuice, and native TypeScript/oxlint tooling.
- Apps: Peekaboo 3.0.0-beta4, ElevenLabsKit 0.1.1, Swabble on Commander 0.2.2.
