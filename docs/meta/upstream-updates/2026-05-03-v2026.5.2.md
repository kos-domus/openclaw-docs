# OpenClaw 2026.5.2

A broad reliability release focused on the npm-first plugin cutover, hot-path performance, UI stability, messaging edge cases, and provider/media correctness.

## Highlights

- External plugin install/update/doctor flows now understand the npm-first cutover better, including stale installs, missing package payloads, and beta-channel fallback.
- Gateway and agent hot paths were trimmed across startup, session listing, task maintenance, prompt preparation, plugin loading, and filesystem guards.
- Control UI and WebChat got resilience fixes across sessions, cron, long-lived websockets, grouped messages, slash-command feedback, and iOS PWA layout.
- Messaging fixes landed across WhatsApp Channel/Newsletter targets, Telegram topic commands and networking, Discord startup/delivery, Slack threads, Signal groups/media, and reply routing.
- Provider/media fixes touched OpenAI-compatible TTS and Realtime, Anthropic-compatible streaming, OpenRouter and DeepSeek replay, LM Studio reasoning metadata, Brave/SearXNG/Firecrawl web search, and voice-call/media paths.

## Breaking or operator-impactful changes

- Plugin lifecycle tooling now assumes the npm-first world more aggressively, so old assumptions about where plugin artifacts live or how updates are resolved are less safe.
- `openclaw gateway restart` now has `--force` and `--wait`, which matters for operators scripting restarts around active tasks.
- Tool planning and plugin loading got more descriptor-first and more cached; custom plugin debugging should now distinguish planning-time visibility issues from runtime execution issues.

## Relevance for this docs repo

The local install is now **one stable release behind** upstream: local `2026.4.29` vs upstream `2026.5.2`.

The highest-value follow-up is a targeted smoke test after upgrade for: plugin management flows, Telegram topic-command behavior, WhatsApp delivery routing, and any MCP/web-search path that depends on Firecrawl or provider-specific media handling.
