# OpenClaw 2026.5.18-beta.1

First checkpoint of the new line, and not a throwaway beta. It front-loads the operator-visible stuff that later shipped in stable: browser dialog control, plugin tooling, restart trace visibility, subagent delivery hardening, and a higher Node floor.

## Highlights

- Browser automation grows a real modal-dialog surface: pending dialog visibility, `blockedByDialog`, and explicit dialog answering by id.
- Plugin authors get a typed simple-tool path with `defineToolPlugin` plus `openclaw plugins build|validate|init`.
- The release keeps leaning into runtime correctness: restart traces, topic-aware Telegram reliability, and fewer ways for subagent completions to disappear into the void.

## Breaking or operator-impactful changes

- Minimum supported Node.js 22 line moves to `22.19`, so old hosts can quietly become upgrade traps.
- New Docker/Podman image args start mattering for package customization, especially if local images were relying on legacy naming only.

## Fixes most worth caring about

- Telegram topic routing and polling keep getting harder to wedge, especially around topic-bound delivery and restart behavior.
- Subagent completion handoff, keep-mode visibility, and requester recovery all get another serious pass.
- Codex/OpenAI paths keep tightening around fail-closed tool policy, OAuth reuse, and context compaction behavior.
