# OpenClaw 2026.5.18

Stable jumped from `2026.5.12` to `2026.5.18`, so this is the one that actually matters for our tracker. The biggest value is not shiny UX — it is operational hardening across browser automation, plugin lifecycle, Telegram/Topic delivery, subagent completion routing, restart behavior, and Codex runtime safety.

## Highlights

- Stable now includes browser dialog handling, typed plugin tooling, richer restart traces, more QA-Lab/runtime parity coverage, and more controlled channel rendering limits.
- The line keeps investing in plugin-first architecture and skills/tooling cleanup without losing focus on live-runtime correctness.
- Android realtime talk mode and Mac Settings redesign shipped too, but they are secondary compared to gateway/channel reliability changes.

## Breaking or operator-impactful changes

- Node.js minimum effectively moves to `22.19` in this line; stale hosts become a real footgun.
- New image-build args (`OPENCLAW_IMAGE_APT_PACKAGES`, `OPENCLAW_IMAGE_PIP_PACKAGES`) matter if local container builds are customized.
- Local CLI is now behind stable by a meaningful jump: `2026.5.7` → `2026.5.18`.

## Fixes most worth caring about

- Telegram got a dense reliability pass: topic-aware queues, topic-preserving media completions, safer polling/restart behavior, and better protection against stale preview/follow-up corruption.
- Subagent delivery got tougher to lose: registry-save-before-accept, completion handoff durability, route projection cleanup, and better requester recovery.
- Gateway/runtime safety improved around config writes, restart ordering, update checks, plugin/channel resolution, and Codex/OpenAI provider routing.
