# OpenClaw 2026.4.2 upstream update summary

## Why it matters

Our installed version is still `2026.3.31`, while upstream has moved to `2026.4.2`.

## Major additions

- `/tasks` adds a chat-native background task board for the current session
- `web_search` gains a bundled SearXNG provider with configurable host support
- Amazon Bedrock Guardrails support lands in the bundled provider
- macOS gets a Voice Wake option for Talk Mode
- Feishu gains dedicated Drive comment-event workflows
- WebChat chat-history truncation becomes configurable
- `agents.defaults.params` adds global default provider params
- Cron jobs can now use per-job tool allowlists via `openclaw cron --tools`
- WhatsApp reaction guidance gains `reactionLevel`
- New bundled Z.AI models: `glm-5.1` and `glm-5v-turbo`

## Fixes most relevant to our setup

- Remote exec approvals are more reliable across Slack, Discord, and Telegram
- Cron/exec approval dead-ends are fixed for isolated automation under trusted policies
- `/model` changes now queue behind active runs instead of interrupting the current turn
- Telegram topic-aware exec approvals stay in the originating topic
- Relative media paths now resolve against the agent workspace
- Bundled plugin runtime staging regressions introduced after `2026.3.31` are fixed
- OpenAI Codex OAuth refresh persistence is fixed, reducing restart loops
- Gateway HTTP handling is more resilient when one request stage fails

## Notable skipped intermediate release

### 2026.4.1-beta.1

- Approval routing improvements
- Stricter cron tool schema handling
- More reliable plugin command session context
- Additional pairing, Matrix, and config validation fixes

## Upgrade note

This looks like a meaningful post-`2026.3.31` release line for our environment, especially for cron autonomy, approvals, session routing, and tool/runtime stability.
